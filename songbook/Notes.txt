Notes from Jennifer Kyrnin's article at
https://www.thoughtco.com/create-epub-file-from-html-and-xml-3467282:

<li><strong>Build your HTML.&nbsp;</strong>Your book is <a href="https://www.thoughtco.com/preformatted-text-3468275" data-component="link" data-source="inlineLink" data-type="internalLink" data-ordinal="1">written in HTML</a>, with CSS for styling. But, it's not just HTML, it's XHTML. So, if you don't normally write in XHTML (closing your elements, using quotes around all attributes, and so on) you will need to convert your HTML to XHTML. You can use one or more XHTML files for your books. Most people separate the chapters into separate XHTML files. Once you have all the XHTML files, put them in a folder all together.</li>
<li><strong>Create a MIME Type File.&nbsp;</strong>In your text editor, open a new document and type: <pre data-type="code">application/epub+zip </pre> Save the file as “mimetype” <em>without any extension</em>. Place that file in the folder with your XHTML files.</li>
<li class="ql-indent-1"><strong>Add your style sheets.&nbsp;</strong>You should create two <a href="https://www.thoughtco.com/what-is-css-3466390" data-component="link" data-source="inlineLink" data-type="internalLink" data-ordinal="2">style sheets</a> for your book one for the pages called<br><pre data-type="code">page_styles.css</pre>: <pre data-type="code">@page {</pre></li>
<li><pre data-type="code">margin-bottom: 5pt;</pre></li>
<li><pre data-type="code">margin-top: 5pt</pre></li>
<li><pre data-type="code">}</pre></li>
<li class="ql-indent-1">Create one for the book styles called <pre data-type="code">stylesheet.css</pre>. You can give them other names, you'll just need to remember what they are. Save these files in the same directory with your XHTML and mimetype files.</li><span class="mntl-sc-block-adslot mntl-sc-block-adslot-inline"></span>
<li><strong>Add your cover image.&nbsp;</strong>Your cover image should be a JPG file no more than 64KB. The smaller you can make it the better, but keep it good looking. <a href="https://www.thoughtco.com/best-image-use-for-kindle-books-3469088" data-component="link" data-source="inlineLink" data-type="internalLink" data-ordinal="3">Small images</a> can be very hard to read, and the cover is where you do your marketing of your book.</li>
<li class="ql-indent-1"><strong>Build your title page.&nbsp;</strong>You don't have to use the cover image as your title page, but most people do. To add your title page, create an XHTML file called <pre data-type="code">titlepage.xhtml</pre>Here is an example of a title page using SVG for the image. Change the highlighted part to point to your cover image:</li>
<li><pre data-type="code">Cover</pre></li>
<li class="ql-indent-1"><strong>Build your "Table of Contents."&nbsp;</strong>Create a file called <pre data-type="code">toc.ncx</pre> in your text editor. This is an XML file, and it should point to all your HTML files in your book. Here is a sample with two elements in the table of contents. Change the highlighted parts to your book, and add additional <pre data-type="code">navPoint</pre> elements for additional sections:</li>
<li><pre data-type="code"><em>How to Build a Website</em></pre></li>
<li><pre data-type="code"><em>Hosting</em></pre></li>
<li><pre data-type="code"><em>Do You Need a Domain Name?</em></pre></li>
<li class="ql-indent-1"><strong>Add a container XML file.&nbsp;</strong>In your text editor, create a file called <pre data-type="code">container.xml</pre> and save it in a sub-directory below your HTML files. The file should read:</li><span class="mntl-sc-block-adslot mntl-sc-block-adslot-inline"></span>
<li class="ql-indent-1"><strong>Create the contents list (</strong><strong></strong><strong></strong><strong></strong><strong><pre data-type="code">content.opf</pre>).</strong> This is the file that explains what your epub book is. It includes <a href="https://www.thoughtco.com/metadata-definition-and-examples-1019177" data-component="link" data-source="inlineLink" data-type="internalLink" data-ordinal="4">metadata</a> about the book (like the author, publish date, and genre). Here is a sample, you should change the parts in yellow to reflect your book:</li>
<li><pre data-type="code">en</pre></li>
<li><pre data-type="code"><em>How to Build a Website</em></pre></li>
<li><em>Jennifer Kyrnin</em></li>
<li>0101-01-01T00:00:00+00:00</li>
<li>0c159d12-f5fe-4323-8194-f5c652b89f5c</li>
<li>That's all the files you need, they should all be in a directory together (except for <pre data-type="code">container.xml</pre>, which goes in a sub-directory <pre data-type="code">META-INF</pre>). We like to then go to the container directory and make sure it has a name that reflects the title and author names.</li>
<li>Once you have the directory of files named how you want it you should use a zip file archive program to zip the directory. My sample directory ends up as a zip file named “How to Build a Website — Jennifer Kyrnin.zip”</li>
<li>Finally, change the file name extension from <pre data-type="code">.zip</pre> to <pre data-type="code">.epub</pre>. Your operating system may protest, but go ahead with it. You want this to have an epub extension.</li><span class="mntl-sc-block-adslot mntl-sc-block-adslot-inline"></span>
<li>Lastly, <strong>test your book.&nbsp;</strong>It's hard to get the epub format correct on the first try, so you should always test your file. Open it in an epub reader like Calibre. And if it doesn't display correctly, you can use Calibre to correct problems.</li>
